# Local Development Setup

This guide will help you set up the GeoChallenge project for local development.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js** 20+ (LTS recommended)
- **npm** 9+ or **pnpm** 8+
- **PostgreSQL** 14+
- **Redis** 6+ (for real-time game sessions)
- **Git** for version control

## Tech Stack Overview

GeoChallenge is a **monorepo** with two main packages:

- **Frontend**: React 18, TypeScript, Vite, TailwindCSS, React Router, Socket.IO Client
- **Backend**: Node.js, Express, TypeScript, Prisma, PostgreSQL, Socket.IO, Redis

## Installation Steps

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/your-org/geochallenge.git
    cd geochallenge
    ```
  </Step>

  <Step title="Install root dependencies">
    Install tooling dependencies (Husky, Prettier, ESLint config):
    
    ```bash
    npm install
    ```
  </Step>

  <Step title="Install frontend dependencies">
    ```bash
    cd frontend
    npm install
    ```
  </Step>

  <Step title="Install backend dependencies">
    ```bash
    cd ../backend
    npm install
    ```
    
    This will automatically run `prisma generate` via the postinstall hook.
  </Step>

  <Step title="Configure environment variables">
    Create a `.env` file in the `backend/` directory:
    
    ```bash
    # Database
    DATABASE_URL="postgresql://user:password@localhost:5432/geochallenge"
    
    # Redis
    REDIS_URL="redis://localhost:6379"
    
    # JWT
    JWT_SECRET="your-super-secret-jwt-key-change-in-production"
    
    # Server
    PORT=3001
    NODE_ENV=development
    
    # Frontend URL (for CORS)
    FRONTEND_URL="http://localhost:5173"
    ```
    
    Create a `.env` file in the `frontend/` directory (optional for local dev):
    
    ```bash
    VITE_API_URL=http://localhost:3001
    VITE_BASE_PATH=/
    ```
  </Step>

  <Step title="Set up the database">
    Create the PostgreSQL database:
    
    ```bash
    createdb geochallenge
    ```
    
    Run Prisma migrations:
    
    ```bash
    cd backend
    npm run db:migrate
    ```
    
    Seed the database with questions:
    
    ```bash
    npm run seed
    ```
  </Step>

  <Step title="Start Redis">
    If using Docker:
    
    ```bash
    docker run -d -p 6379:6379 redis:7-alpine
    ```
    
    Or start your local Redis server:
    
    ```bash
    redis-server
    ```
  </Step>

  <Step title="Start the development servers">
    In one terminal, start the backend:
    
    ```bash
    cd backend
    npm run dev
    ```
    
    In another terminal, start the frontend:
    
    ```bash
    cd frontend
    npm run dev
    ```
    
    The frontend will be available at `http://localhost:5173` and the backend API at `http://localhost:3001`.
  </Step>
</Steps>

## Verify Installation

Once both servers are running:

1. Open `http://localhost:5173` in your browser
2. You should see the GeoChallenge home page
3. Register a new account to test authentication
4. Check the backend health endpoint: `http://localhost:3001/health`

## Development Workflow

### Running Tests

**Frontend tests:**
```bash
cd frontend
npm run test          # Unit tests with Vitest
npm run test:watch    # Watch mode
npm run test:e2e      # End-to-end tests with Playwright
```

**Backend tests:**
```bash
cd backend
npm run test          # Unit tests with Vitest
npm run test:watch    # Watch mode
```

### Linting and Formatting

**From the root directory:**
```bash
npm run lint          # Lint frontend + backend
npm run format        # Format all files with Prettier
npm run format:check  # Check formatting without writing
```

**Frontend quality gate (used in CI):**
```bash
cd frontend
npm run ci:quality    # Runs lint + test + build
```

### Pre-commit Hooks

The project uses Husky and lint-staged to automatically:
- Run ESLint on staged `.ts`/`.tsx` files
- Format code with Prettier
- Prevent commits with linting errors

## Common Issues

### Database Connection Errors

If you see `Can't reach database server`, verify:
- PostgreSQL is running: `pg_isready`
- DATABASE_URL is correct in `.env`
- Database exists: `psql -l | grep geochallenge`

### Redis Connection Errors

If the backend fails to start with Redis errors:
- Check Redis is running: `redis-cli ping` (should return `PONG`)
- Verify REDIS_URL in `.env`

### Port Already in Use

If port 3001 or 5173 is already in use:
- Change `PORT` in `backend/.env`
- Change `server.port` in `frontend/vite.config.ts`

### Prisma Client Issues

If you see `@prisma/client did not initialize yet`:
```bash
cd backend
npm run db:generate
```
