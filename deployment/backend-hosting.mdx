## Overview

The GeoChallenge backend is deployed to [Render](https://render.com) using a declarative `render.yaml` configuration file. This includes the Node.js API server with WebSocket support, PostgreSQL database, and Redis cache.

## Architecture

The backend consists of:

- **Web Service** (Node.js): API + WebSocket server
- **PostgreSQL Database**: Managed by Render (external)
- **Redis Cache**: Managed by Render (external)
- **Static Site** (optional): Frontend can also be deployed to Render

## Render Configuration

The entire infrastructure is defined in `render.yaml` at the repository root.

### Backend API Service

```yaml
services:
  - type: web
    name: geochallenge-api
    runtime: node
    region: oregon
    plan: free
    rootDir: backend
    buildCommand: npm install && npm run build
    startCommand: npx prisma migrate deploy && npm start
```

### Service Configuration Details

| Setting | Value | Description |
|---------|-------|-------------|
| **type** | `web` | HTTP service with public URL |
| **runtime** | `node` | Node.js environment |
| **region** | `oregon` | US West Coast datacenter |
| **plan** | `free` | Free tier (spins down after inactivity) |
| **rootDir** | `backend` | Working directory |
| **buildCommand** | `npm install && npm run build` | Installs deps and compiles TypeScript |
| **startCommand** | `npx prisma migrate deploy && npm start` | Runs migrations then starts server |

<Warning>
  The **free plan** spins down after 15 minutes of inactivity. The first request after spin-down takes 30-60 seconds (cold start).
</Warning>

## Build Process

<Steps>
  <Step title="Install Dependencies">
    ```bash
    npm install
    ```
    Installs all production and dev dependencies from `package.json`.
  </Step>
  <Step title="Compile TypeScript">
    ```bash
    npm run build
    ```
    Compiles TypeScript to JavaScript in `dist/` directory.
  </Step>
  <Step title="Deploy Migrations">
    ```bash
    npx prisma migrate deploy
    ```
    Applies pending Prisma database migrations to PostgreSQL.
  </Step>
  <Step title="Start Server">
    ```bash
    npm start
    ```
    Starts the production server on `PORT=3001`.
  </Step>
</Steps>

## Environment Variables

The backend requires these environment variables on Render:

```yaml
envVars:
  - key: NODE_ENV
    value: production
  - key: DATABASE_URL
    sync: false  # Set manually
  - key: REDIS_URL
    sync: false  # Set manually
  - key: JWT_SECRET
    generateValue: true  # Auto-generated secure value
  - key: JWT_EXPIRES_IN
    value: 7d
  - key: FRONTEND_URL
    sync: false  # Set manually
  - key: PORT
    value: 3001
```

### Required Manual Configuration

You must manually set these in the Render dashboard:

- **DATABASE_URL**: PostgreSQL connection string
- **REDIS_URL**: Redis connection string  
- **FRONTEND_URL**: Your frontend URL (for CORS)

See [Environment Variables](/deployment/environment-variables) for details.

<Warning>
  **JWT_SECRET** is auto-generated by Render. Never use the default `default-secret-change-me` in production.
</Warning>

## Deployment Methods

### Option 1: Render Dashboard (Recommended)

<Steps>
  <Step title="Connect Repository">
    1. Go to [Render Dashboard](https://dashboard.render.com)
    2. Click **New +** → **Blueprint**
    3. Connect your GitHub repository
  </Step>
  <Step title="Configure Blueprint">
    Render will automatically detect `render.yaml` and parse the configuration.
  </Step>
  <Step title="Set Environment Variables">
    In the Render dashboard, set the required environment variables:
    - DATABASE_URL (from your Render PostgreSQL service)
    - REDIS_URL (from your Render Redis service)
    - FRONTEND_URL (your GitHub Pages or Render static site URL)
  </Step>
  <Step title="Deploy">
    Click **Apply** to deploy all services.
  </Step>
</Steps>

### Option 2: Render CLI

```bash
# Install Render CLI
npm install -g @render/cli

# Login
render login

# Deploy from render.yaml
render blueprint deploy
```

## Database Setup

Create a PostgreSQL database on Render:

<Steps>
  <Step title="Create Database">
    1. In Render Dashboard, click **New +** → **PostgreSQL**
    2. Name it `geochallenge-db`
    3. Select **Free** plan
  </Step>
  <Step title="Copy Connection String">
    Copy the **Internal Database URL** from the database info page.
  </Step>
  <Step title="Set DATABASE_URL">
    Paste the connection string into your web service's `DATABASE_URL` environment variable.
  </Step>
</Steps>

The Prisma migration will run automatically on deployment:

```bash
npx prisma migrate deploy
```

## Redis Setup

Create a Redis instance on Render:

<Steps>
  <Step title="Create Redis">
    1. In Render Dashboard, click **New +** → **Redis**
    2. Name it `geochallenge-redis`
    3. Select **Free** plan (25 MB)
  </Step>
  <Step title="Copy Connection String">
    Copy the **Internal Redis URL** from the Redis info page.
  </Step>
  <Step title="Set REDIS_URL">
    Paste the connection string into your web service's `REDIS_URL` environment variable.
  </Step>
</Steps>

## Keep Backend Awake

To minimize cold starts on the free tier, the project includes a GitHub Actions workflow that pings the backend every 10 minutes:

**Workflow**: `.github/workflows/keep-backend-awake.yml`

```yaml
on:
  schedule:
    - cron: '*/10 * * * *'  # Every 10 minutes
```

### Setup

<Steps>
  <Step title="Get Health Endpoint">
    Your backend health endpoint will be:
    ```
    https://geochallenge-api.onrender.com/health
    ```
  </Step>
  <Step title="Configure GitHub Secret">
    1. Go to your GitHub repo → **Settings** → **Secrets and variables** → **Actions**
    2. Click **New repository secret**
    3. Name: `BACKEND_HEALTHCHECK_URL`
    4. Value: Your health endpoint URL
  </Step>
  <Step title="Verify">
    The workflow will run every 10 minutes automatically.
  </Step>
</Steps>

<Warning>
  If `BACKEND_HEALTHCHECK_URL` is not set, the workflow will skip with a warning instead of failing.
</Warning>

## Troubleshooting

### Build Failed

1. Check Render logs for TypeScript or dependency errors
2. Verify `package.json` scripts are correct
3. Test build locally:
   ```bash
   cd backend
   npm install
   npm run build
   ```

### Migration Failed

1. Check that `DATABASE_URL` is set correctly
2. Verify database is accessible from the service
3. Check Prisma schema for errors
4. Run migration locally to test:
   ```bash
   npx prisma migrate deploy
   ```

### Service Won't Start

1. Check that all required environment variables are set
2. Verify `JWT_SECRET` is not the default value
3. Check Redis and PostgreSQL connection strings
4. Review startup logs in Render dashboard

### Cold Starts

On the free tier, services spin down after 15 minutes of inactivity:

- **Solution 1**: Upgrade to paid tier ($7/month) for always-on service
- **Solution 2**: Use the keep-awake workflow (already configured)
- **Solution 3**: Accept 30-60s delay on first request
