---
title: Get Game History
api: GET /api/game/history
description: Retrieve the authenticated user's recent game history
---

## Authentication

Required. Must provide a valid JWT token:

```
Authorization: Bearer <token>
```

## Query Parameters

<ParamField query="limit" type="number" optional default="10">
  Maximum number of games to return
  
  - Defaults to 10 if not specified
  - Capped at a reasonable maximum by the server
</ParamField>

## Response

<ResponseField name="games" type="array">
  Array of game result objects, ordered by most recent first
  
  <ResponseField name="id" type="string">
    Game result identifier (CUID)
  </ResponseField>
  
  <ResponseField name="score" type="number">
    Total score achieved in the game
  </ResponseField>
  
  <ResponseField name="correctCount" type="number">
    Number of correct answers
  </ResponseField>
  
  <ResponseField name="totalQuestions" type="number">
    Total questions in the game (typically 10)
  </ResponseField>
  
  <ResponseField name="category" type="string" optional>
    Question category if a specific category was selected
    
    Possible values: `FLAG`, `CAPITAL`, `MAP`, `SILHOUETTE`, `MIXED`
  </ResponseField>
  
  <ResponseField name="gameMode" type="string">
    Type of game played
    
    Possible values:
    - `SINGLE` - Solo play
    - `DUEL` - Real-time 1v1
    - `CHALLENGE` - Asynchronous challenge
  </ResponseField>
  
  <ResponseField name="createdAt" type="string">
    ISO 8601 timestamp of when the game was completed
  </ResponseField>
</ResponseField>

## Error Responses

<ResponseField name="401 Unauthorized">
  Missing or invalid JWT token
</ResponseField>

<ResponseField name="500 Internal Server Error">
  Database or server error
</ResponseField>

## Implementation Details

From `backend/src/controllers/game.controller.ts:198` and `backend/src/services/game.service.ts`:

- Games are returned in descending order by `createdAt`
- Only games belonging to the authenticated user are returned
- The `details` field (answer-by-answer breakdown) is excluded for performance
- Use the limit parameter to implement pagination in your UI

<Info>
This endpoint is useful for displaying a user's game history on their profile page or for tracking progress over time.
</Info>
